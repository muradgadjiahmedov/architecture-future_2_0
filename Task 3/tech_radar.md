# Task3 — Технологический радар «Будущее 2.0»

## Adopt (внедрять)
| Технология / подход | Назначение | Поддерживаемые бизнес-сценарии |
|---|---|---|
| Cloud Object Storage (Data Lake) | Масштабируемое хранение данных | Быстрая отчётность, рост объёмов данных |
| Lakehouse (cloud SQL engine) | Аналитика и витрины данных | Self-service аналитика, ad-hoc отчёты |
| Event Bus (Kafka / PubSub) | Асинхронные доменные события | Независимое развитие доменов, интеграция новых бизнесов |
| Semantic Layer | Единые KPI и метрики | Управленческие решения, консистентность отчётов |
| IAM / RBAC / ABAC | Управление доступом | Регуляторика, безопасность данных |

## Trial (пилотировать)
| Технология / подход | Назначение | Поддерживаемые бизнес-сценарии |
|---|---|---|
| Data Mesh (Data Products) | Доменные витрины данных | Масштабирование экосистемы, автономия команд |
| Data Catalog & Lineage | Governance и доверие к данным | Self-service, снижение ошибок аналитики |
| Data Quality Framework | Контроль качества данных | Качество медицинских и фин. сервисов |
| MLOps | Управление жизненным циклом ML | Надёжность ИИ-сервисов |

## Hold (временно оставить)
| Технология | Причина | Ограничения |
|---|---|---|
| Power BI | Зрелость для бизнеса | Минимизировать кастомизации |
| Apache Camel | Уже используется | Постепенная замена на event-driven |
| Legacy DWH (SQL Server 2008) | Критичные процессы | Вывод логики, только read-only |

## Retire (выводить)
| Технология | Причина | Эффект для бизнеса |
|---|---|---|
| PowerBuilder | Технологический долг | Снижение затрат на поддержку |
